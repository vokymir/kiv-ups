Vsechny mozne *validni* pruchody klienta serverem popsany:
- nepopisuji se PING/PONGy
- pokud prijde jina zprava nez ve scenarich, bude automaticky ukoncena komunikace
- nektere zpravy nevyzaduji odpoved a mohou byt opakovany vicekrat napr. kvuli update na serveru. zapis: <od>: <zprava> /repeat/<how many times>/

1. server je plny
C: wants to connect
S: refuse

2. doje do lobby
C: wants to connect
S: accept
-- Protocol communication begins --
S: Configuration (nastav si klienta podle serveru)
S: Want Nick
C: Nick
S: Rooms /repeat/inf/
...

3. vybere si roomku
...
S: Rooms /repeat/inf/
C: Join room (x) / Create room
S: Room /repeat/inf/
...

4. opusti roomku
...
S: Room /repeat/inf/
C: Leave Room
S: Rooms /repeat/inf/
...

5. zacne hra v roomce
...
S: Room /repeat/inf/
S: Game Snapshot (= zacala hra, klient dostal snapshot)
S: Played/Drew/Passed /repeat/n players - 1/
S: Play
C: Play (card)/Draw/Pass
S: Played/Drew/Passed /repeat/n players - 1/
...

6. konec hry v roomce
...
S: Played/Drew/Passed /repeat/n active players/ OR C: Play (card)/Draw/Pass
S: Game Finished (= posle leaderboard)
(delay, dokud user neodklikne ze chce do lobby)
C: Go To Lobby
S: Rooms /repeat/inf/
...

7. kratke odpojeni ve hre
...
S: Game Snapshot OR S: Played/Drew/Passed /repeat/n active players/ OR C: Play (card)/Draw/Pass
S: Brief Disconnect (player X)
...
S: Reconnect (player X)
...

7.5 pohled pro odpojeneho klienta
...
S: Game Snapshot OR S: Played/Drew/Passed /repeat/n active players/ OR C: Play (card)/Draw/Pass
... trying to reconnect ...
S: Game Snapshot
(could be S: Play)
...

8. definitivni odpojeni ve hre
...
S: Brief Disconnect (player X)
...
S: Terminal Disconnect (player X)
(could be S: Game Finished)
...
